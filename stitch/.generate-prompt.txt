# Alışkanlık Takip Uygulaması — PRD

## 1. Genel Bakış

Bu uygulama, kullanıcıların günlük alışkanlıklarını ekleyip takip edebileceği basit, hızlı ve kullanıcı dostu bir web uygulamasıdır. Hedef kitle: günlük rutinlerini dijital ortamda organize etmek isteyen bireyler. Arayüz tamamen Türkçe olup, tek sayfada (SPA) çalışır. Veriler tarayıcı localStorage'ında saklanır; sunucu veya veritabanı gerektirmez.

## 2. Hedefler

1. Kullanıcıların saniyeler içinde yeni alışkanlık eklemesini sağlamak
2. Tamamlanan alışkanlıkları görsel olarak (streak, işaret) ödüllendirmek
3. localStorage ile tarayıcı kapansa bile veriyi korumak
4. Mobil ve masaüstü cihazlarda sorunsuz çalışmak (responsive)
5. Erişilebilirlik standartlarına (WCAG 2.1 AA) uymak
6. İlk yükleme süresini 2 saniyenin altında tutmak

## 3. Tech Stack Detayı

- Framework: React 18 (fonksiyonel bileşenler, hooks)
- Build: Vite (HMR, hızlı derleme, optimize edilmiş bundle)
- Dil: TypeScript (strict mode)
- Styling: Tailwind CSS (utility-first, responsive)
- State: React useState + useReducer (küçük ölçekli, global state yöneticisine gerek yok)
- Storage: localStorage (persisted state)
- Routing: Yok — tek sayfa uygulama
- Icon Library: Lucide React
- Font: Inter (Google Fonts, body + heading)

## 4. Fonksiyonel Gereksinimler

### 4.1 Alışkanlık Ekleme
- Kullanıcı bir input alanına alışkanlık adını yazar
- Max 100 karakter
- Zorunlu alan: boş string kabul edilmez
- Hata: "Lütfen bir alışkanlık adı girin" (Türkçe, input altında kırmızı)
- Ekle butonu: "Ekle" (primary, yeşil ton)
- Başarı durumu: Input temizlenir, liste anında güncellenir, localStorage'a yazılır

### 4.2 Alışkanlık Listeleme
- Eklenen tüm alışkanlıklar kart/liste şeklinde gösterilir
- Sıralama: en son eklenen üstte
- Her satırda: alışkanlık adı, tamamlandı işareti, sil butonu
- Tamamlananlar görsel olarak farklı görünür (üstü çizili, soluk renk)

### 4.3 Tamamlandı İşaretleme
- Her alışkanlığın solunda bir checkbox/toggle
- Tıklanınca durum değişir: tamamlandı / tamamlanmadı
- Tamamlananlar hemen görsel güncellenir
- localStorage anında sync edilir
- Streak mantığı yok (tek gün için basit toggle)

### 4.4 Alışkanlık Silme
- Her satırın sağında "Sil" butonu (secondary, kırmızı ton, küçük)
- Tıklanınca onay sormadan silinir (basit uygulama, confirmation yok)
- Animasyon: yumuşak fade-out (Tailwind transition)
- localStorage'dan da silinir

### 4.5 Boş Durum (Empty State)
- Hiç alışkanlık yoksa ekranda:
  - İllüstrasyon yerine Lucide "ListChecks" ikonu (büyük, soluk)
  - Mesaj: "Henüz alışkanlık eklemediniz."
  - Alt metin: "Yukarıdaki alana yazarak ilk alışkanlığınızı ekleyin."

### 4.6 localStorage Hatası
- localStorage yazma/okuma hatası durumunda:
  - Kırmızı banner: "Veriler kaydedilemedi. Tarayıcı ayarlarınızı kontrol edin."
  - Uygulama çalışmaya devam eder (state memory'de tutulur)

## 5. Veri Modeli

```typescript
interface Habit {
  id: string;           // crypto.randomUUID()
  title: string;        // max 100 karakter
  completed: boolean;   // default false
  createdAt: number;    // Date.now(), sıralama için
}
```

### localStorage Şeması
- Key: `habit-tracker-data`
- Value: `Habit[]` JSON string
- Migrate: yok (basit yapı, versiyonlama gerekmez)

## 7. Non-Functional Requirements

### 7.1 Performans
- İlk yükleme < 1.5 saniye (Vite + küçük bundle)
- localStorage okuma/yazma senkron, UI bloklanmaz
- Liste 1000+ öğeye kadar sorunsuz (React virtualize gerekmez)
- Bundle size hedefi: < 150 KB (gzip)

### 7.2 Erişilebilirlik (WCAG 2.1 AA)
- Klavye navigasyonu: Tab ile input → buton → liste → toggle → sil
- Checkbox'larda ARIA label: `"{title} alışkanlığını tamamla"`
- Sil butonunda ARIA label: `"{title} alışkanlığını sil"`
- Kontrast: tüm metinler 4.5:1 üzeri
- Focus state: `focus:ring-2 focus:ring-emerald-500 focus:outline-none`
- Touch hedefleri: buton ve checkbox min 44x44 px

### 7.3 Tarayıcı Desteği
- Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- iOS Safari 14+, Android Chrome 90+
- Responsive: 320px (mobil) → 1920px (masaüstü)
- Layout: max-width 640px (container), ortalanmış
